# 系统消息页面 PRD

## 1. 功能概述

### 1.1 背景

系统消息是平台与用户沟通的重要渠道，用于通知用户账户变动、系统公告、用量提醒等信息。

### 1.2 目标

- 让用户能够查看所有系统消息
- 提供消息分类筛选能力
- 支持消息已读/未读状态管理
- 在导航栏显示未读消息数量红点

### 1.3 用户价值

| 场景 | 价值 |
|------|------|
| 账户安全 | 及时知晓密码修改、登录异常等安全事件 |
| 系统公告 | 了解平台更新、维护通知 |
| 用量提醒 | 掌握 Token 消耗情况，避免超限 |

---

## 2. 消息类型定义

| 类型 | msg_type | 图标 | 说明 | 示例 |
|------|----------|------|------|------|
| 账户通知 | `account` | 🔐 | 账户相关的安全和状态变更 | 密码已修改、邮箱已验证 |
| 系统公告 | `notice` | 📢 | 平台级别的通知 | 系统维护、新功能上线 |
| 用量提醒 | `usage` | 📊 | Token 用量相关提醒 | 用量达到 80%、已超限 |

## 3. 页面结构

### 3.1 入口

#### 入口 A：Header 铃铛图标（主入口）

- 位置：Header 右侧
- 显示未读数量红点
- **Hover 悬浮面板**：鼠标悬停显示消息缩略列表
- **点击**：进入完整消息列表页

#### 入口 B：用户菜单（辅助入口）

- 位置：侧边栏左下角用户头像区域
- 点击弹出向上悬浮菜单
- 菜单包含"系统消息"选项，显示未读数量

## 5. API 对接

### 5.1 接口列表

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/messages` | GET | 获取消息列表 |
| `/api/v1/messages/:id` | GET | 获取消息详情 |
| `/api/v1/messages/:id/read` | PUT | 标记单条已读 |
| `/api/v1/messages/read-all` | PUT | 标记全部已读 |
| `/api/v1/messages/unread-count` | GET | 获取未读数量 |

### 5.2 请求参数

**GET /api/v1/messages**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | 否 | 页码，默认 1 |
| page_size | int | 否 | 每页数量，默认 20，最大 100 |
| msg_type | string | 否 | 消息类型筛选：account/notice/usage |

### 5.3 响应格式

**消息列表响应**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "messages": [
      {
        "id": 1,
        "title": "密码修改成功",
        "content": "您的账户密码已成功修改...",
        "msg_type": "account",
        "is_read": false,
        "created_at": "2024-02-02T14:30:00Z",
        "read_at": null
      }
    ],
    "total": 25,
    "page": 1,
    "page_size": 20
  }
}
```

**未读数量响应**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "count": 3
  }
}
```

## 7. 状态管理

### 7.1 Store 设计

```typescript
// src/store/useMessageStore.ts
interface MessageState {
  // 数据
  messages: SystemMessage[];
  unreadCount: number;

  // 分页
  page: number;
  hasMore: boolean;

  // 筛选
  activeTab: 'all' | 'account' | 'notice' | 'usage';

  // 加载状态
  isLoading: boolean;
  isRefreshing: boolean;
  isLoadingMore: boolean;

  // Actions
  fetchMessages: (refresh?: boolean) => Promise<void>;
  fetchUnreadCount: () => Promise<void>;
  markAsRead: (id: number) => Promise<void>;
  markAllAsRead: () => Promise<void>;
  setActiveTab: (tab: string) => void;
}
```

### 7.2 数据流

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  Messages   │────▶│   Store     │────▶│    View     │
│    API      │     │ (Zustand)   │     │  (React)    │
└─────────────┘     └─────────────┘     └─────────────┘
                           │
                           ▼
                    ┌─────────────┐
                    │ AsyncStorage│
                    │  (可选缓存)  │
                    └─────────────┘
```

### 7.3 未读数量同步

需要在以下时机更新未读数量：

1. **App 启动时**：调用 `fetchUnreadCount()`
2. **进入消息页时**：刷新列表和未读数
3. **标记已读后**：本地减少 count 或重新获取
4. **定时轮询**：可选，每 5 分钟检查一次

## 10. 边界情况

### 10.1 异常处理

| 场景 | 处理方式 |
|------|----------|
| 网络错误 | 显示错误提示，提供重试按钮 |
| 消息不存在 | 显示 Toast "消息不存在" |
| Token 过期 | 自动刷新 Token 或跳转登录 |
| 标记已读失败 | Toast 提示，保持原状态 |

### 10.2 性能优化

- 使用 FlatList 虚拟列表渲染
- 消息内容预览截断（最多 100 字符）
- 图片/富文本内容暂不支持，仅显示纯文本
- 本地缓存消息列表（可选）
